## 一、服务器端操作步骤

在服务器 (10.1.3.32) 上执行以下步骤：

### 步骤1：拉取最新代码

```bash
cd /path/to/pms-project
git pull origin main
```

### 步骤2：构建前端（如服务器上有 Node 环境）

```bash
cd yudao-ui-admin-vue3
pnpm install
pnpm build:local
```

### 步骤3：部署静态资源

将构建产物 `dist/` 目录的内容复制到 Nginx 的静态文件目录：

```bash
# 假设 Nginx 静态资源路径
cp -r dist/* /usr/share/nginx/html/pms/
# 或
cp -r dist/* /var/www/pms/
```

### 步骤4：重载 Nginx

```bash
nginx -s reload
```

---

## 二、使用预构建包部署（推荐）

如果服务器上没有 Node 环境，直接将 `pms-dist.tar.gz` 传输到服务器解压：

```bash
# 在服务器上
scp user@build-server:/workspace/pms-dist.tar.gz /tmp/
cd /tmp
tar -xzf pms-dist.tar.gz
cp -r dist/* /usr/share/nginx/html/pms/
nginx -s reload
```

## 三、新增菜单配置（数据库操作）

本次修复新增了「字典管理」页面，需要在数据库中插入菜单记录：

```sql
-- 在 pms 菜单表中新增字典管理菜单
INSERT INTO system_menu (name, permission, type, path, component, parent_id, sort, visible, status)
VALUES ('字典管理', 'pms:dict:list', 2, 'dict', 'pms/dict/index', 
        (SELECT id FROM system_menu WHERE name = '系统配置'), 10, 1, 0);
```
